## 2. 상세 기능 차이점

### 2.1 대시보드 기능

#### ✅ 동일한 기능
- 페이지 접속 (`/ai-bot` 경로)
- 헤더 표시 ("AI 알림봇" 제목 및 설명)
- 탭 네비게이션 (대시보드/설정)
- 업종 선택 카드 4개 (쇼핑몰, 예약, 프랜차이즈, 매장)
- 쇼핑몰 활성화, 나머지 "준비중" 표시
- 템플릿 그리드 10개 표시
- 기본 템플릿 5개 자동 선택 (ID: 1,2,3,4,5)
- 템플릿 선택/해제 기능
- "기본" 배지 표시
- 템플릿 저장 버튼
- 디버그 로그 뷰어 통합

#### ⚠️ 차이점
- **없음** - 모든 기능 동일하게 구현됨

---

### 2.2 설정 기능

#### ✅ 동일한 기능
- 설정 탭 접속
- 결제 섹션 표시
- 연동 섹션 표시 (스마트스토어)
- 템플릿 선택 섹션 표시
- 스마트스토어 연동 활성화 체크박스
- API Key, API Secret, Store ID 입력 필드
- 연동 설정 저장 버튼
- 템플릿 체크박스 리스트 (10개)
- 템플릿 선택/해제 기능
- 템플릿 선택 저장 버튼

#### ⚠️ 차이점

**결제 설정 기능**:
- **배포 버전**: 
  - 현재 잔액 표시 ✅
  - 충전 금액 선택 (1만원, 3만원, 5만원, 10만원, 30만원, 50만원) ✅
  - 직접 입력 필드 ✅
  - 결제 수단 선택 (신용카드, 계좌이체, 가상계좌) ✅
  - 충전하기 버튼 ✅
  - 충전 내역 테이블 ✅
  - **포트원 연동 기능** ✅

- **신규 버전 (cursorAI)**:
  - `PaymentSettings.vue` 컴포넌트 존재 ✅
  - UI 구조는 동일하게 구현됨 ✅
  - **포트원 연동 로직 미구현** ⚠️
  - 실제 결제 처리 기능 없음 ⚠️

**차이점 요약**:
- 신규 버전은 결제 설정 UI만 구현되어 있고, 실제 포트원 연동 및 결제 처리 로직은 미구현 상태입니다.

## 5. 미구현 기능

### 5.1 결제 기능

**배포 버전**:
- ✅ 포트원 연동 완료
- ✅ 실제 결제 처리 가능
- ✅ 충전 내역 저장 및 조회

- ⚠️ 결제 설정 UI만 구현

- ❌ 실제 결제 처리 불가능
- ❌ 충전 내역 기능 미구현

**구현 필요 사항**:
1. 포트원 SDK 연동
2. 결제 API 엔드포인트 구현
3. 결제 내역 저장/조회 기능
4. 잔액 관리 기능
